# Epic 4 retrospective — Containerize, gate coverage, ship assessor evidence

**Epic:** 4 — Docker/Compose, coverage gate, QA pack, BMAD + AI narrative docs  
**Date (UTC):** 2026-05-13  
**Sprint tracking:** `_bmad-output/implementation-artifacts/sprint-status.yaml`

## What went well

- **Incremental delivery:** Dockerfiles (4.1–4.2) landed before Compose (4.3), keeping risk bounded.  
- **Profile-based Compose:** **`postgres`** default path preserved for inner-loop dev; **`full`** profile isolates heavier container runs.  
- **Coverage + contract:** Vitest v8 reports and CI gate (NFR-09) surfaced a real **`PATCH done`** validation gap (**AJV coercion**) — fixed with explicit **`coerceTypes: false`**.  
- **Assessor docs:** Single **`docs/qa/index.md`** plus **`docs/bmad-implementation-narrative.md`** and **`docs/ai-integration-log.md`** give reviewers a short trail without duplicating the PRD.

## Challenges / risks

- **Docker CLI fragmentation:** `docker compose` vs `docker-compose` vs **`COMPOSE_PROFILES`** — documented in compose header + README; still a support burden.  
- **Port collisions:** Full stack uses host **3000** / **9080** — conflicts with host **`npm run dev:api`**; README calls this out.  
- **Lint vs generated coverage:** HTML under **`web/coverage/`** required ESLint **`globalIgnores`**.

## Lessons (systems, not blame)

1. **Document the lowest-common-denominator CLI** next to the “happy path” command.  
2. **Run coverage with the same `DATABASE_URL` as CI** before merging infra stories.  
3. **Ignore generated folders early** when adding coverage to front-end repos.
