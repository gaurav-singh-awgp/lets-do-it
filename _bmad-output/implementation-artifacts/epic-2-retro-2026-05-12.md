## Challenges and System Improvements

### 1. `last_updated` timestamp regressed in every single story

Every code review (2.1, 2.2, 2.3, 2.4) patched a backward `last_updated` in `sprint-status.yaml`. The dev agent writes this field manually on every save and consistently back-dates it. Four reviews caught it; four patches applied. Root cause: no guard or automation.

**Improvement (B3):** Automate the field via script or remove it entirely. Manual timestamps in a machine-edited file are a broken process.

### 2. Playwright infrastructure fragile — now documented but not fixed

Story 2.4 encountered missing Chromium, an arm64/x64 browser-cache mismatch requiring a manual symlink, and E2E flaking under parallel execution due to a shared API/DB. Workaround: `workers: 1` in `playwright.config.ts`. This works today but will be a bottleneck in Story 3.4's CI matrix (parallel lint + unit + E2E + axe jobs).

**Improvement (B1 + B2):** Address the two Epic 1 carry-forwards (A1 and A4) as a single small prep commit before Story 3.3:
- B1: `if (process.env.CI) return;` guard in Playwright `globalSetup`
- B2: Vitest serial integration project in config; document `workers: 1` rationale in `playwright.config.ts`

### 3. A1 and A4 from Epic 1 carried forward twice

Both items are small. Both came back as friction in Epic 2. They're now B1 and B2 with explicit scheduling before E2E-heavy stories (3.3, 3.4).
